# Tests

Tests in Iris are intelligent workflows that define how agents should interact with and validate your web application. They combine AI-powered navigation with specific validation criteria to ensure your application works as expected.

## Test Structure

### Components
- **Instructions**: Natural language descriptions of test objectives
- **Validation Points**: Specific elements or states to verify
- **Success Criteria**: Conditions that determine test passage
- **Agent Configuration**: Specific settings for test execution

## Types of Tests

### 1. User Journey Tests
- Simulate complete user workflows
- Validate multi-step processes
- Test complex user interactions

### 2. Functional Tests
- Feature validation
- Input/output verification
- Error handling checks

### 3. Visual Tests
- UI component verification
- Layout validation
- Responsive design testing

## Test Creation

### Manual Creation
1. Define test objectives
2. Specify test steps
