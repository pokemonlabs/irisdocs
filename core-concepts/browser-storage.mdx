# Browser Storage

Iris QA lets you configure browser storage to handle authentication and user state, enabling automated testing of authenticated workflows without manual login steps.

## Benefits

- Skip repetitive logins by reusing auth tokens
- Test protected features directly 
- Maintain consistent test states

## Setup

1. Go to Projects dashboard
2. Select your project
3. Open Settings
4. Find Browser Storage section

[Screenshot placeholder for browser storage settings]

## Storage Options

### Cookies
Common auth storage for:
- JWT tokens
- Session/auth cookies

### localStorage 
For persistent data:
- Access/refresh tokens
- User info

### sessionStorage
For temporary data:
- Session tokens
- Temp credentials
